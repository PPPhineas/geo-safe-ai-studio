> 本文整理灾圈风险报告**第三章「分类统计」**的图表要素、数据来源与技术栈，供渲染层实现。
> 配套：[`灾圈监测点风险研判_报告成稿模板.md`](灾圈监测点风险研判_报告成稿模板.md)（第三章）、[`空间分析内容与技术栈.md`](空间分析内容与技术栈.md)。
> 原则：**图由代码生成**（与"数字交代码"一致），与报告统计数字同源；配色与空间分布图呼应。
>
> **渲染选型（已定）**：全报告统一 **Plotly + kaleido**（与空间分布图同一套视觉语言）。kaleido 静态导出自带渲染、不依赖系统浏览器。

## 二、各图表关键说明

### 3.1 灾害类型分布
- 类别 ≤5 用饼图较直观；类别多（含"未分类"）用**水平柱状图**避免饼图拥挤；
- 类型配色与空间分布图中点位的"形状/颜色编码"保持一致，便于图文对照。

### 3.2 规模等级分布
- `scale` 是**有序类别**：横轴固定按 小 → 中 → 大 → 特大 排列，空值归入"未分级"并显示；
- 不要按计数大小排序，否则失去等级递进含义。

### 3.3 预警等级分布
- 用**预警语义配色**（红/橙/黄/蓝），并与第四章地图点位的预警颜色严格一致——全报告同一套预警色板；
- 含"未定级"类别。

### 3.4 隐患识别情况
- 环形图或单条占比图展示 `is_hidden_danger=1` 比例；
- 标注绝对数 + 百分比（数值来自代码）。

### 3.5 威胁要素汇总
- 威胁人数、户数（量级相近）可同轴；财产（万元）量纲差异大，建议**双 y 轴**或**单独成图**；
- 柱顶直接标数值，避免读图误差。

## 三、技术栈汇总

| 角色 | 库 | 必需/可选 |
|---|---|---|
| 图表绘制 | **plotly**（`graph_objects` / `make_subplots`） | 必需 |
| 静态导出（PNG/SVG，自带渲染、免系统浏览器） | **kaleido** | 必需 |
| 中文字体 | 设 `layout.font.family="Microsoft YaHei"` 即可（基本无乱码坑） | 必需 |
| 数据整理 | **pandas**（分组结果转绘图结构） | 必需 |

## 四、通用规范与易踩的坑

1. **中文字体**：Plotly 设 `layout.font.family="Microsoft YaHei"` 即可，基本无 matplotlib 那种乱码坑；kaleido 用系统字体渲染，确保服务器装有该中文字体。
2. **统一色板与模板**：预警等级红橙黄蓝、灾害类型配色 + Plotly `template`（如 `plotly_white`），**全报告（图表 + 地图）共用一套**，保证图文一致。
3. **数值标签**：`text=` + `textposition`，数值来自代码统计，不二次计算；饼图用 `textinfo` 显数量+百分比。
4. **空值类别可见**：未分级/未定级/未分类等类别要显示并计数，不能静默丢弃（与缺失计数呼应，进数据局限说明）。
5. **有序类别排序**：规模、预警按等级序排列（饼图 `sort=False`、柱图固定 x 顺序），非按频次。
6. **统一尺寸/缩放**：所有图表统一 `width/height`，导出 `scale=2` 提升清晰度，嵌报告排版整齐；输出 PNG（嵌入）+ SVG（矢量备用）。
