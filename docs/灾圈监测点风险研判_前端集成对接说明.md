# 灾圈监测点风险研判 · 前端集成对接说明（桌面端）

> 本文是「灾圈监测点地质灾害风险研判」功能模块接入上层 **AI Studio** GUI 工作台（桌面端 Shell）的对接规范。

## 3. API 契约 逐字段

挂载前缀 `/api/v1`。

| 端点 | 方法 | 用途 |
|---|---|---|
| `/api/v1/risk-report` | POST | 同步：一次性返回完整报告（降级用） |
| `/api/v1/risk-report/stream` | POST | **SSE 流式（本模块主用）** |
| `/api/v1/risk-report/{report_id}/bundle` | GET | 下载离线 zip |
| `/static/figures/{report_id}/{key}.png` | GET | 图件静态资源 |
| `/health` | GET | 存活探针 `{"status":"ok"}` |
| `/api/v1/a2ui/action` | POST | A2UI 交互回传（可选） |

### 3.1 请求体 `RiskReportRequest`

```jsonc
{
  "geometry": {
    "kind": "polygon",                 // 字面量，固定 "polygon"
    "coordinates": [[lon, lat], ...]   // WGS84 经纬度顶点，≥3 个；注意是 (经度, 纬度) 顺序
  },
  "report_id": "可选：外部报告编号；省略则后端生成 12 位十六进制"
}
```

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `geometry.kind` | `"polygon"` | 是 | 当前仅支持多边形 |
| `geometry.coordinates` | `[ [number, number], ... ]` | 是 | WGS84 `[经度, 纬度]`，≥3 顶点 |
| `report_id` | `string` | 否 | 省略由后端生成 |

### 3.2 响应体 `RiskReportResponse`（同步端点 & SSE `done` 事件）

| 字段                | 类型              | 说明                                                                                         |
| ----------------- | --------------- | ------------------------------------------------------------------------------------------ |
| `report_id`       | `string`        | 报告编号                                                                                       |
| `point_count`     | `int`           | 圈内有效监测点数（已剔除 `is_hex=1`）。**为 0 时走简化报告**，见 [§6](<#6. 报告渲染与产物>)                                 |
| `report_markdown` | `string`        | 渲染后的报告正文（成稿模板拼装，含图片占位）                                                                     |
| `judgement`       | `object`        | AI 研判 JSON（已过校验），结构见 [§3.3](<#3.3 judgement 结构 用于结构化卡片展示>)                                    |
| `figures`         | `{ key: url }`  | 图件标识 → **相对** URL（如 `map_figure`、`chart_type_distribution`、`chart_warning_distribution` 等） |
| `attachments`     | `{ name: url }` | 附件名 → **相对** URL（点位过多时的完整清单 CSV、连片带 CSV 等）                                                 |

> 空结果分支（`point_count==0`）只保证返回 `report_id` / `point_count` / `report_markdown`；`judgement` / `figures` / `attachments` 可能为空对象。

### 3.3 judgement 结构 用于结构化卡片展示

`report_markdown` 已把研判文字拼进正文，但前端若要做**结构化卡片**（而非纯 Markdown），可直接消费 `judgement`：

```jsonc
{
  "overall_risk":  { "level": "string", "basis": "string" },
  "dominant_hazard": { "type": "string", "cause_analysis": "string", "source_note": "string" },
  "common_induce_factors": ["string", ...],
  "trend": { "judgment": "string", "evidence_points": ["monitor_point_code", ...] },
  "key_points": [ { "monitor_point_code": "string", "reason": "string", "suggestion": "string" } ],
  "recommendations": { "urgent": ["string"], "near_term": ["string"], "routine": ["string"] },
  "data_limitations": ["string", ...]   // DEM 未接入、缺失字段、校验降级等说明
}
```

> 数字一律以后端代码值为准；`judgement` 内**不含**前端需要再计算的数字。`key_points[].monitor_point_code` 保证落在圈内点集合内（后端第 1 道校验）。涉及地形的表述会带「（基于填报值，未经地形数据校验）」字样，原样展示即可。

### 3.4 A2UI 界面协议 SSE a2ui 事件 流式专用

研判通过校验后、渲染图件前，后端通过 SSE `a2ui` 事件下发一份**声明式 UI 组件列表**，前端可直接按 `type` 渲染对应 UI 组件，构建结构化报告视图。`RiskReportResponse` 中**不含**此字段，仅 SSE 流式路径可用。

**顶层结构 `A2uiSurface`**：

```jsonc
{
  "surfaceId": "risk-abc123",           // 界面标识
  "dataModel": {                        // 共享数据：report_id、point_count
    "report_id": "abc123",
    "point_count": 42
  },
  "components": [                       // 扁平组件列表，无嵌套
    { "id": "risk_card", "type": "card", "props": { ... } },
    { "id": "chart_type_distribution", "type": "chart", "props": { ... } },
    // ... 更多组件
  ]
}
```

**6 种泛化基元 `type` 与各 `props` 契约**：

| `type` | 组件 id（固定） | `props` 字段 | 说明 |
|---|---|---|---|
| `card` | `risk_card` | `{ title, fields: [{label, value}] }` | 整体风险等级卡片 |
| `card` | `hazard_card` | `{ title, fields: [{label, value}] }` | 主要灾害类型卡片（含成因、数据来源） |
| `chart` | `chart_type_distribution` | `{ chartType: "pie", title, data: [{name, value}] }` | 灾害类型分布饼图 |
| `chart` | `chart_scale_distribution` | `{ chartType: "bar", title, categories, series: [{name, data}] }` | 规模分布柱状图 |
| `chart` | `chart_warning_distribution` | `{ chartType: "bar", title, categories, series }` | 预警等级分布柱状图 |
| `chart` | `chart_hidden_danger_ratio` | `{ chartType: "pie", title, data }` | 隐患识别占比饼图 |
| `chart` | `chart_threat_summary` | `{ chartType: "bar", title, categories, series }` | 威胁要素汇总柱状图（人数/户数/财产） |
| `list` | `key_points` | `{ columns: [{key, label}], rows: [{code, reason, suggestion}], maxDisplay }` | 重点关注点位列表（columns+rows 模式） |
| `list` | `recommendations` | `{ groups: [{label, key, items: string[]}] }` | 建议措施分组列表（groups 模式） |
| `map` | `chart_map` | `{ center: [lon, lat], zoneBoundary?, clusterHulls?, hotspotGrid? }` | 空间分布地图：中心点 + 灾圈边界 + 连片带凸包 + 热力网格 |
| `text` | `text_limitations` | `{ content, format: "markdown" }` | 数据局限说明（Markdown 格式） |
| `image` | — | `{ src, alt?, width? }` | 静态图片嵌入（当前后端暂未生成，预留给后续图件直出） |

> **组件含义**：`id` 是组件唯一标识（后端固定，见上表），`type` 决定前端渲染哪个 UI 组件，`props` 是该组件的全部数据。组件列表无序但语义分组：卡片在上 → 图表居中 → 列表/地图/文本在下。
> 有数据则出组件，无数据自动跳过（如 `point_count==0` 时不发出 `a2ui`）。列表组件的 `maxDisplay` 表示默认折叠前展示条数。

**A2UI 交互回传**（可选，非必须）：

若前端需把用户交互（如点击某条重点点位「已标记」）回传给后端做二次研判，可 POST `/api/v1/a2ui/action`：

```jsonc
{                                               // A2uiActionRequest
  "report_id": "abc123",
  "surfaceId": "risk-abc123",
  "action": "click_detail",                     // 动作类型
  "componentId": "key_points",                  // 触发组件 id
  "payload": { "rowIndex": 0 }                  // 附加数据
}
```

当前阶段后端仅回 `{"status":"acknowledged"}`，后续可接 agent 二次研判、报告更新等。

## 4. SSE 流式对接 本模块重点

### 4.1 关键约束：必须用 `fetch` 流式解析，不能用 `EventSource`

`POST /api/v1/risk-report/stream` 是 **POST + 请求体**；而浏览器原生 `EventSource` **只支持 GET、无法带 body**。因此前端必须用 `fetch` + `ReadableStream` 手动解析 `text/event-stream`。

后端已设 `Cache-Control: no-cache`、`Connection: keep-alive`、`X-Accel-Buffering: no`（抑制反代缓冲）。

### 4.2 事件帧格式

标准 SSE 帧，事件间以空行（`\n\n`）分隔：

```
event: stage
data: {"stage":"selection","msg":"初筛:从 ClickHouse 取灾圈内监测点…"}

event: a2ui
data: {"surfaceId":"risk-abc123","components":[...],"dataModel":{...}}

event: done
data: { ...完整 RiskReportResponse... }
```

### 4.3 事件类型 UI 映射
| `event` | `data` | UI 处理 |
|---|---|---|
| `stage` | `{stage, msg, ...}` | 进度步进条：初筛 → 清洗 → 统计 → 空间 → 研判 → 校验 → 渲染 |
| `reasoning` | `{text}` | AI 思考流增量（可折叠展示，追加拼接） |
| `content` | `{text}` | 研判 JSON 输出增量（可不展示，或展示原始流） |
| `validation` | `{ok, errors}` | 校验结果提示；`ok=false` 时后端会重试或降级 |
| `a2ui` | `A2uiSurface`（见 [§3.4](<#3.4 A2UI 界面协议 SSE a2ui 事件 流式专用>)） | 声明式 UI 界面意图：研判完成后、渲染前发出。收到后按组件 type 渲染对应 UI 组件，构建结构化报告视图（替代/补充 Markdown 渲染）。空结果分支（`point_count==0`）不发出 |
| `warn` | `{msg}` | 非致命黄条（如某图件渲染失败降级，不阻塞报告） |
| `done` | 完整 `RiskReportResponse` | 拿到最终结果，切换到报告视图 |
| `error` | `{msg}` | 致命错误红条，终止本次 |
> 事件流以 `done`（成功）或 `error`（失败）收尾。收到任一即停止读取。1

### 4.4 参考解析 伪码

```ts
async function streamRiskReport(req, { onEvent, signal }) {
  const resp = await fetch(new URL("/api/v1/risk-report/stream", API_BASE_URL), {
    method: "POST",
    headers: { "Content-Type": "application/json", "Accept": "text/event-stream" },
    body: JSON.stringify(req),
    signal,
  });
  if (!resp.ok || !resp.body) throw new Error(`HTTP ${resp.status}`);

  const reader = resp.body.getReader();
  const decoder = new TextDecoder();
  let buf = "";
  for (;;) {
    const { value, done } = await reader.read();
    if (done) break;
    buf += decoder.decode(value, { stream: true });

    let sep;
    while ((sep = buf.indexOf("\n\n")) !== -1) {     // 按空行切完整事件
      const frame = buf.slice(0, sep);
      buf = buf.slice(sep + 2);

      let event = "message", dataLines = [];
      for (const line of frame.split("\n")) {
        if (line.startsWith("event:")) event = line.slice(6).trim();
        else if (line.startsWith("data:")) dataLines.push(line.slice(5).trimStart());
      }
      const data = JSON.parse(dataLines.join("\n"));
      onEvent(event, data);                          // 分发到 §4.3 的 UI 处理
      if (event === "done" || event === "error") return data;
    }
  }
}
```

### 4.5 中断 / 重连 / 降级

- **中断**：用户取消用 `AbortController.abort()`，后端会随连接断开停止。
- **不做自动重连**：报告生成不可幂等续传，失败整次重试（重新 POST）。
- **降级到同步**：若桌面壳网络栈或中间反代仍缓冲流（进度长时间不动），回退调用同步 `POST /api/v1/risk-report`，前端转圈等待最终 `RiskReportResponse`。建议 client 暴露一个 `stream: boolean` 开关。

## 5. 灾圈选区交互 地图组件

两种入参方式，二选一或并存（都只产出 `geometry.coordinates`，对后端契约无影响）：

- **方式 A · GeoJSON 上传（参考实现已有）**：用户传 `.geojson`，前端取 `Polygon` 外环 / `MultiPolygon` 第一个面的外环转成 `coordinates`。逻辑见 [`tests/ui/index.html`](../tests/ui/index.html) 的 `extractPolygon()`，可直接搬。适合「已有灾圈矢量文件」的场景。
- **方式 B · 地图交互画多边形（工作台推荐升级）**：用 **MapLibre GL JS**（与后端 `go.Scattermap` / MapLibre 同一视觉语言），配 `@mapbox/mapbox-gl-draw`（或 `maplibre-gl-draw`）。底图用 `open-street-map` / carto 系，**免 token**。绘制结果输出 WGS84 `[经度, 纬度]` 顶点列表，组装进 `geometry.coordinates`（≥3）。
- **前端校验**（提交前）：
  - 顶点数 ≥ 3；
  - 多边形自相交时提示用户修正；
  - **避开 ~(0,0) 脏坐标区**：数据源存在坐标默认落在约 (0,0) 的脏点，选区跨到该区会把脏点圈进来导致结果异常；
  - 数据合理经度区间约 **100–110°E**，顶点落在区间外给软提示（不强制拦截）。
- 「生成报告」按钮触发 [§4](<#4. SSE 流式对接 本模块重点>) 的 SSE 调用。

## 6. 报告渲染与产物

- **Markdown 渲染**：用 `react-markdown` + GFM 插件（`remark-gfm`，报告含表格）。
- **图片 URL 重写**：`report_markdown` 内嵌的图片是相对 `/static/...`。渲染前统一把 `/static` 前缀拼成 `API_BASE_URL` 的绝对地址（预处理替换，或在 `react-markdown` 的 `img` 组件里 `new URL(src, API_BASE_URL)`）。`figures` 字典提供同样的相对 URL，可用于结构化展示或预加载。
- **附件**：`attachments`（点位过多时的完整清单 CSV、连片带 CSV）渲染为下载链接（同样拼绝对 URL）。
- **离线包**：「下载离线包」按钮 → `GET /api/v1/risk-report/{report_id}/bundle` 得到 zip（`report.md` 内图片链接已改为相对 `figures/`，含全部图件 + 附件 + 研判 JSON）。落盘方式因框架而异：**Electron** → 主进程 `dialog.showSaveDialog()` + `fs.writeFile`（通过 `ipcRenderer.invoke`）；**Tauri** → `@tauri-apps/plugin-dialog` + `@tauri-apps/plugin-fs`。解压后离线打开 `report.md` 即含图，不再依赖在线 `/static`。
- **空结果**：`point_count==0` 时后端**不调模型、不渲染图件**，返回一份简化提示 Markdown（说明圈内无有效监测点）。前端正常把它当普通 Markdown 渲染即可，**不要当错误处理**。

- **A2UI 结构化渲染**（推荐，SSE 流式专用）：收到 `a2ui` 事件后，遍历 `components` 数组，按 `type` 分发渲染：
  - `card` → `<A2uiCard>`：渲染标题 + 键值对字段列表；
  - `chart` → `<A2uiChart>`：按 `props.chartType`（`"pie"` / `"bar"`）选 ECharts 对应图型，`data` / `categories+series` 直接驱动；
  - `list` → `<A2uiList>`：有 `columns+rows` 用表格模式，有 `groups` 用分组列表模式，`maxDisplay` 控制折叠阈值；
  - `map` → `<A2uiMap>`：用 MapLibre GL JS 渲染散点/凸包/热力，`center` 定视图中心；
  - `text` → `<A2uiText>`：`format: "markdown"` 时用 `react-markdown` 渲染；
  - `image` → `<img>`：`src` 需拼 `API_BASE_URL` 绝对地址。
  - A2UI 渲染完成后，Markdown 报告（`report_markdown`）可作为**附录/详情**降级展示，也可两视图切换。
  - `dataModel` 是共享上下文，可在组件间透传（如 `report_id` 用于后续 bundle 下载）。

## 7. 桌面壳工程注意点（Electron / Tauri，前端落地清单）

核心逻辑（fetch/SSE/渲染）与壳框架无关，以下按框架分列差异项：

| 事项 | Electron | Tauri v2 |
|---|---|---|
| **网络请求** | 渲染进程直接用 `fetch`，内置 Chromium | WebView `fetch`；也可用 `@tauri-apps/plugin-http`（需在 capabilities 配置 `http:default` + scope 允许列表） |
| **CORS / Origin** | 开发：`http://localhost:PORT`；生产：`file://` 或自定义 protocol。`file://` 下建议 `webSecurity: false` | Origin 为 `tauri://localhost`（macOS/Linux）或 `http://tauri.localhost`（Windows）；后端 `allow_origins=["*"]` 即通 |
| **CSP** | 在 `BrowserWindow` 的 `webPreferences` 中设 CSP，放行 `connect-src` / `img-src` 至 `API_BASE_URL` + 瓦片源 | 在 `tauri.conf.json` → `app.security.csp` 中放行同上 |
| **离线包落盘** | `ipcRenderer.invoke` → 主进程 `dialog.showSaveDialog()` + `fs.writeFile` | `@tauri-apps/plugin-dialog`（保存对话框）+ `@tauri-apps/plugin-fs`，对应 capabilities 授权 |
| **进程通信** | `ipcMain.handle` / `ipcRenderer.invoke` 通道注册 | Tauri command（Rust 端 `#[tauri::command]`）或 plugin API |

> 共性：`fetch` + SSE 解析逻辑跨框架完全复用，**不需要任何条件编译**。差异仅在于上述壳层 API（CSP 配置位置、文件落盘调用方式），且都收敛在少数几个 API client / 工具函数中。

- ⚠️ **底图联网依赖**：后端 kaleido 静态导出地图时需联网拉底图瓦片；**内网无外网环境下导出的地图底图会空白**（点位/连片带/热点等几何仍在）。请在运维说明中提示：内网部署需自备瓦片源或接受底图空白。

## 8. 功能模块在工作台中的形态

作为 AI Studio 壳层注册的一个**路由 / 功能面板**，建议双栏布局：

```
┌──────────────────────────────────────────────────────────────┐
│  灾圈监测点风险研判                                    [生成报告] │
├───────────────────────────┬──────────────────────────────────┤
│                           │  ● 初筛 ✓  ● 清洗 ✓  ● 统计 …      │  ← stage 步进条
│   MapLibre 地图            │  ┌────────────────────────────┐   │
│   · 画多边形选灾圈          │  │ AI 研判思考流（可折叠）        │   │  ← reasoning
│   · 顶点校验 / 脏区提示      │  └────────────────────────────┘   │
│                           │  ─────────────────────────────    │
│                           │  报告视图（Markdown + 图件）         │  ← done 后
**建议封装的 API client（收敛 base-url 拼接 / SSE 解析 / URL 重写 / A2UI 渲染）：**

```ts
createRiskReport(req): Promise<RiskReportResponse>          // 同步端点（降级）
streamRiskReport(req, { onEvent, signal }): Promise<RiskReportResponse>  // SSE（含 a2ui 事件）
getBundleUrl(reportId): string                              // 拼 bundle 下载 URL
absStatic(relUrl): string                                   // 相对 → 绝对
postA2uiAction(reportId, surfaceId, action, componentId, payload?): Promise<Ack>  // 交互回传
```

**错误码约定**

| HTTP | 含义 | 前端处理 |
|---|---|---|
| `200` | 成功 | 正常渲染 |
| `404` | bundle 不存在（report_id 错误/已清理） | 提示重新生成 |
| `500` | 链路异常 | 展示响应体 `detail`（形如 `类型: 信息`） |
| `501` | 后端 stub 未实现（**开发期**，业务逻辑尚未填实现） | 提示「后端该阶段未实现」，不当致命错误 |

> 当前后端处于 stub 阶段：业务调用会抛 `NotImplementedError` → 返回 `501`。前端可先按本契约把 UI/交互打通，后端逐阶段填实现后即自动转 `200`。
