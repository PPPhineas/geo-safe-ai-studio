# 空间分析内容与技术栈（空间分布图）

> 本文整理灾圈风险报告中**空间分布图**的各分析要素（图层）、对应分析方法与技术栈，供渲染层实现。
> 配套：[`灾圈监测点风险研判_报告成稿模板.md`](灾圈监测点风险研判_报告成稿模板.md)（第四章空间格局）、[`灾圈监测点风险研判_配套装配与渲染说明.md`](灾圈监测点风险研判_配套装配与渲染说明.md)。
> 复用原则：空间分析**算一次**，结果既喂 AI 研判文字（`cluster_summary`/`hotspot_summary`/`affected_extent_summary`），又用于出图。
>
> **渲染选型（已定）**：全报告统一 **Plotly + kaleido**。架构为「**分析归分析、渲染归 Plotly**」——DBSCAN/凸包/缓冲/KDE 等 GIS 计算仍由 sklearn/scipy/shapely/geopandas 完成，把算出的**几何**交给 Plotly 渲染，与统计图表同一套视觉语言。

## 一、图层要素 × 分析方法 × 技术栈

图层自下而上叠放：

> 技术栈分两列：**分析**（计算几何）与**渲染**（Plotly 画图）。

| 图层      | 数据来源                    | 分析方法（计算几何）                                                  | 渲染（Plotly）                                                  |
| ------- | ----------------------- | ----------------------------------------------------------- | ----------------------------------------------------------- |
| ① 底图    | 在线瓦片                    | —                                                           | **Plotly map** `style="open-street-map"`/carto（**免 token**） |
| ② 影响范围  | 圈内点集                    | 凸包 / alpha-shape / 缓冲并集（**shapely**，可选 **alphashape**）      | `Scattermap` lines + `fill="toself"`                        |
| ③ 风险热点  | 点位坐标（可按威胁人数/财产加权）       | KDE 或 Getis-Ord Gi\*（**scipy/sklearn**；或 **esda+libpysal**） | `Densitymap` 热力 / 分级着色点                                     |
| ④ 连片隐患带 | 点位坐标                    | DBSCAN → 每簇凸包/alpha-shape（**scikit-learn + shapely**）       | `Scattermap` 填充多边形                                          |
| ⑤ 灾圈边界  | 灾圈几何（多边形，已转坐标系） | 多边形直接用（**shapely**）                                 | `Scattermap` lines + 半透明填充                                  |
| ⑥ 监测点位  | 圈内点 经纬度 + 属性            | 按类型/预警/规模分类符号化                                              | `Scattermap` markers（颜色编预警、大小编规模、hover 显编号）                 |
| ⑦ 制图要素  | —                       | —                                                           | Plotly 图例/标题；比例尺/指北针按需自绘                                    |

> 绘制顺序（trace 叠加先后）：底图 → 影响范围（半透明）→ 热点 → 连片带 → 灾圈边界 → 点位。点位最后加，置于最上层保证可读。
>
> **Plotly 6.x 用 `go.Scattermap`/`go.Densitymap`（MapLibre）**，旧 `Scattermapbox` 已弃用。

## 二、各分析要素关键说明

### ② 影响范围
- 点少 → 凸包（convex_hull）；
- 分布不规则、要贴合形状 → alpha-shape；
- 强调单点影响半径 → 各点缓冲(buffer)并集。
- 三选一，依业务对"影响范围"的定义而定。

### ③ 风险热点
- **KDE**：连续热力面，直观；
- **Getis-Ord Gi\***：统计显著的冷热点，更严谨；
- 可用 `threaten_population` / `threaten_assets` 作权重，使威胁大的点权重更高。

### ④ 连片隐患带
- DBSCAN 必须在**投影坐标系**下做：`eps` 单位米（业务定，如 500m），`min_samples`（如 2~3）；
- 簇内点再取凸包 / alpha-shape 生成"带状"包络面；
- 与 `cluster_summary` 同源，复用同一次聚类结果。

### ⑥ 监测点位（信息密度核心）
建议多维符号化，与第三章统计图呼应：
- **颜色** → 预警等级 `warning_level`（与图表预警色板同一套）
- **大小** → 规模 `scale` / 威胁程度
- **灾害类型** → Plotly 地图 marker 形状受限，建议**按类型分 trace**（图例可点选）或并入 hover；不要强行用形状编码
- **hover** → 显示 `monitor_point_code` 等明细，与报告溯源一致

## 三、整体技术栈汇总

**分析层（计算几何）**

| 角色 | 库 | 必需/可选 |
|---|---|---|
| 空间数据容器、投影 | **geopandas** | 必需 |
| 几何运算（缓冲/凸包/并集） | **shapely** | 必需 |
| 坐标转换 / 投影 | **pyproj** | 必需 |
| 聚类（连片带） | **scikit-learn** | 必需 |
| 核密度 / 空间运算 | **scipy** | 必需 |
| Gi\* 冷热点 | **esda + libpysal** | 可选（需统计显著性时） |
| alpha-shape 包络 | **alphashape** | 可选 |

**渲染层（出图）**

| 角色 | 库 | 必需/可选 |
|---|---|---|
| 地图渲染 + 底图（免 token，`open-street-map`/carto） | **plotly**（`go.Scattermap`/`go.Densitymap`） | 必需 |
| 静态导出 PNG/SVG（自带渲染，无需系统浏览器） | **kaleido** | 必需 |

## 四、坐标系 / 渲染注意点

- **分析阶段**（DBSCAN、缓冲、面积、距离）须在**投影坐标系**（CGCS2000 高斯 / UTM）下进行，单位米才准确；
- **渲染阶段** Plotly 地图直接吃 **WGS84 经纬度**（不要传投影坐标）——即"投影坐标系算几何 → 转回经纬度交 Plotly 画"；
- **底图免 token**：用 `style="open-street-map"` 或 carto 系，无需 Mapbox token；
- ⚠️ **静态导出需渲染时联网**拉瓦片：内网无外网环境底图会空白（点位/几何仍在）。应对：内网搭瓦片服务 / 离线 mbtiles，或接受无底图浅色背景；
- **输出**：PNG（嵌报告，`scale=2` 提升清晰度）+ SVG（矢量备用）。
