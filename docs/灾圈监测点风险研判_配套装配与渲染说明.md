# 灾圈监测点风险研判 — 配套装配与渲染说明

> 本文是 [`灾圈监测点风险研判提示词.md`](灾圈监测点风险研判提示词.md) 的配套文档，说明：
> 1. 占位符如何从查询结果装配（三层装配规则）
> 2. 字段清洗规则（应对 NULLABLE / STRING 类型字段）
> 3. 重点点位筛选规则
> 4. 渲染层校验与报告拼装
> 5. 研判 JSON → 报告章节映射
>
> 数据源表：`dwd_gh_v1.dwd_monitor_point_info_view`（监测点信息表）。
> 整体链路：CH 空间筛选 → GeoPandas 空间分析 + 代码统计 → 三层装配 → AI 研判 → 报告渲染。

## 一、字段清洗规则（装配前置）

表中大量字段为 `NULLABLE(STRING)`，数值类信息被存成字符串，装配前必须统一清洗，否则统计报错或偏差。

| 字段 | 原始类型 | 清洗规则 |
|---|---|---|
| `threaten_population` 威胁人数 | NULLABLE(STRING) | 去单位/空白 → 转 int；空/非法 → 计为 0，并记入"缺失计数" |
| `threaten_residents` 威胁户数 | NULLABLE(STRING) | 同上 |
| `threaten_assets` 威胁财产 | DECIMAL(18,2) | 已是数值，空值按 0 计并记缺失 |
| `avg_slope` 平均坡度 | NULLABLE(STRING) | 提取数值，剔除"度/°"；异常（<0 或 >90）→ 标记需复核，不参与统计 |
| `elevation` 高程 | NULLABLE(STRING) | 提取数值；仅用于属性分带，不当地形分析 |
| `hazard_*` 长/宽/厚/体积、`watershed_area`、`gully_length` | NULLABLE(STRING) | 提取数值，空值跳过；仅作填报值引用 |
| `scale` 规模等级 | NULLABLE(STRING) | 归一到 {小,中,大,特大}；其它/空 → "未分级" |
| `warning_level` 预警等级 | NULLABLE(STRING) | 归一到统一等级集；空 → "未定级" |
| `monitor_point_type` 灾害类型 | NULLABLE(STRING) | 归一到 {滑坡,崩塌,泥石流,地面塌陷,...}；空 → "未分类" |
| `is_hex` 是否核销 | UINT8 | **筛选阶段即过滤 is_hex=1**，不进入任何统计 |
| `is_hidden_danger` 隐患识别 | NULLABLE(UINT8) | 1=隐患；空按 0 计 |

> 原则：**所有"空值/非法值被如何处理"都要可追溯**，缺失计数最终进入 `data_limitations`，让研判知道哪些统计建立在不完整数据上。

## 二、三层装配规则（占位符生成）

### 第一层 + 第二层：既定事实统计（代码算，对应 prompt 第二、三部分）

| 占位符 | 生成规则 |
|---|---|
| `zone_geometry_desc` | 多边形：`N 顶点多边形，外接范围…` |
| `zone_area` | 多边形几何面积（投影坐标系下，单位 km²/m²） |
| `point_count` | 圈内 `is_hex=0` 的点数 |
| `type_distribution` | 按 `monitor_point_type` 分组计数，降序，格式 `滑坡 12、崩塌 5…` |
| `scale_distribution` | 按 `scale` 分组计数（含"未分级 N"） |
| `warning_level_distribution` | 按 `warning_level` 分组计数（含"未定级 N"） |
| `hidden_danger_ratio` | `is_hidden_danger=1` 计数 / 总数，格式 `8 个（占 32%）` |
| `threaten_population_total` | 清洗后 `threaten_population` 求和 |
| `threaten_residents_total` | 清洗后 `threaten_residents` 求和 |
| `threaten_assets_total` | 清洗后 `threaten_assets` 求和（万元） |

### 空间分析结论（GeoPandas，对应 prompt 第三部分）

| 占位符 | 生成规则 |
|---|---|
| `cluster_summary` | DBSCAN 对圈内点坐标聚类（投影坐标系，eps 按业务定，如 500m）；输出连片带数量及各带成员 `monitor_point_code` 列表 |
| `hotspot_summary` | 核密度 / Getis-Ord Gi\*；输出高值热点区位置描述及涉及点位 |
| `affected_extent_summary` | 点集凸包 / alpha-shape / 缓冲；输出影响范围描述 |

### 第三层：语义素材抽取（代码抽取，对应 prompt 第五部分）

| 占位符 | 生成规则 |
|---|---|
| `common_induce_factors` | 对 `induce_factors` 做分词/关键词频次统计，取 Top-N 高频项 |
| `common_lithology` | 对 `lithology` 归类计数，取主要岩性 |
| `typical_status_excerpts` | 从 `current_status` 选取代表性描述（优先重点点位 + 趋势词命中"加剧/扩大/变形"等），节选拼接，控制长度 |

## 三、重点点位筛选规则（生成 `key_points_detail`）

只把"重点点位"放入明细，其余仅进统计，以控制上下文长度。命中**任一**条件即为重点：

1. `warning_level` 属高等级（如红/橙，按业务等级定义）；
2. `scale` ∈ {大, 特大}；
3. `threaten_population` 或 `threaten_assets` 居前（如 Top-N 或超阈值）；
4. 落在 `cluster_summary` 的任一连片带内；
5. `is_hidden_danger=1` 且伴随趋势恶化描述。

每条明细字段（固定顺序，便于模型解析）：
```
monitor_point_code | monitor_point_name | monitor_point_type | scale |
warning_level | threaten_population | threaten_assets | lithology |
avg_slope(填报) | induce_factors | current_status(原文)
```

> 点位数量上限建议设阈值（如 ≤30 条），超出时按风险优先级截断，并在 `data_limitations` 注明"明细仅列前 N 个高风险点"。

## 四、渲染层校验与拼装

研判返回 JSON 后，**渲染前**由代码侧执行校验，不依赖模型自觉：

1. **编号溯源校验**：递归提取 JSON 中所有 `monitor_point_code`，与圈内点集合比对；存在集合外编号 → 判不合格 → 回退重生成（或剔除该条并记警告）。
2. **来源标注校验**：`dominant_hazard.source_note` 及 `cause_analysis` 中地形相关表述需含"基于填报值，未经地形数据校验"字样；缺失 → 判不合格。
3. **数字一致性**：报告正文的统计数字**一律用代码值填充**；模型文字中复述的数字仅作叙述，最终以代码值覆盖校对。
4. **JSON 结构校验**：字段齐全、类型正确；缺字段 → 回退重生成。
5. **空结果处理**：`point_count=0` 时不调用模型，直接渲染"灾圈内无有效监测点"提示报告。

## 五、研判 JSON → 报告章节映射

| 报告章节 | 数据来源 |
|---|---|
| 一、概述（灾圈范围、点位总数） | 代码：`zone_*` / `point_count` |
| 二、监测点清单（带编号，可追溯） | 代码：圈内点集（重点点位标注） |
| 三、分类统计（类型/规模/预警/威胁） | 代码：第二层统计 + 图表 |
| 四、空间格局（连片带/热点/影响范围） | 代码：GeoPandas + 图（matplotlib/contextily） |
| 五、综合风险研判 | AI：`overall_risk` + `dominant_hazard` + `common_induce_factors` + `trend` |
| 六、重点关注点位 | AI：`key_points`（编号已校验） |
| 七、分级处置建议 | AI：`recommendations`（urgent/near_term/routine） |
| 八、数据局限说明 | AI：`data_limitations` + 代码侧缺失计数 |
| 附录 | 代码：完整点位数据表 |

> 报告中**所有数字来自代码统计、所有研判文字来自 AI**，二者拼装但来源可区分，全程可核查。

## 六、未接入 DEM 的边界声明（固定写入报告与研判）

本方案当前未接入数字高程模型(DEM)：

- 空间分析（连片/热点/影响范围/属性统计）**不受影响，照常准确**；
- 涉及坡度/坡向/汇水/沟道的内容**均来自填报值，未经地形数据校验**；
- 无法独立校验填报地形值的正确性，相关研判须标注来源、必要时提示现场复核；
- 后续若接入公开 DEM（SRTM 30m / ALOS 12.5m，建议 ≤10m），可补充坡度/汇水计算与填报值校验，提升研判客观性——列为改进项。
