# 灾圈风险研判前端对接成果交接

本文面向前端对接，整理当前后端已经完成且可直接使用的能力。详细字段说明仍以 [`灾圈监测点风险研判_前端集成对接说明.md`](./灾圈监测点风险研判_前端集成对接说明.md) 为准；本文用于快速开工和验收。

## 1. 当前已具备的后端能力

当前 FastAPI 后端已经跑通完整风险研判链路：

```text
灾圈多边形
  -> 圈内监测点筛选
  -> 字段清洗 / 区域补全
  -> 统计分析 / 空间分析
  -> 时序接入 / 趋势预测
  -> 降雨-变形耦合 / 预警历史
  -> LLM 研判 JSON
  -> A2UI 组件
  -> Markdown 报告 / PNG 图件 / CSV 附件 / zip 离线包
```

已完成并可对接的产物：

| 产物 | 状态 | 前端用途 |
|---|---|---|
| 同步报告接口 | 已完成 | 作为流式失败时的降级方案 |
| SSE 流式报告接口 | 已完成 | 主对接方式，展示阶段进度、AI 输出、A2UI、最终报告 |
| A2UI 组件输出 | 已完成 | 前端用 ECharts / MapLibre / 原生组件渲染结构化报告 |
| Markdown 成稿 | 已完成 | 报告正文预览、归档展示 |
| PNG 图件 | 已完成 | 最终报告、离线包、A2UI 降级显示 |
| 报告版本查询 | 已完成 | 历史版本恢复和切换 |
| 批注修订 | 已完成 | 前端段落批注后生成新版 Markdown |
| 离线 zip 包 | 已完成 | 下载报告、图件、附件、研判 JSON |
| 本地参考 UI | 已完成 | `http://127.0.0.1:8000/ui/` |

## 2. 前端最小接入路径

推荐前端先按下面顺序接入：

1. 配置 `API_BASE_URL`，所有 API、`/static` 图片、附件 URL 都通过它拼绝对地址。
2. 实现灾圈输入，输出 WGS84 `[lon, lat]` 顶点数组。
3. 用 `fetch + ReadableStream` 调 `POST /api/v1/risk-report/stream`，不要用 `EventSource`。
4. 按 SSE 事件分发 UI：`stage` 展示进度，`a2ui` 渲染结构化卡片/图表/地图，`done` 渲染最终 Markdown。
5. 对 `report_markdown` 中的 `/static/...` 图片地址做绝对路径重写。
6. 接入 `GET /api/v1/risk-report/{report_id}/bundle` 下载离线包。
7. 再接版本查询与批注修订。

本仓库已有纯前端参考实现：[`tests/ui/index.html`](../tests/ui/index.html)。它已经覆盖 GeoJSON 输入、SSE 解析、图件展示、Markdown 预览、离线包下载，可直接迁移关键逻辑。

## 3. 接口清单

接口统一挂载在 `/api/v1`。

| 方法 | 路径 | 说明 |
|---|---|---|
| `POST` | `/api/v1/risk-report/stream` | 主接口，SSE 流式生成报告 |
| `POST` | `/api/v1/risk-report` | 同步生成报告，作为降级方案 |
| `GET` | `/api/v1/risk-report/{report_id}` | 获取报告版本列表和当前版本正文 |
| `POST` | `/api/v1/risk-report/{report_id}/revise` | 根据段落批注生成新版 Markdown |
| `GET` | `/api/v1/risk-report/{report_id}/bundle?version_id=...` | 下载离线 zip 包 |
| `POST` | `/api/v1/a2ui/action` | A2UI 交互回传，当前返回 ack |
| `GET` | `/health` | 服务健康检查 |

请求体：

```json
{
  "geometry": {
    "kind": "polygon",
    "coordinates": [[104.1, 30.6], [104.2, 30.6], [104.2, 30.7]]
  },
  "report_id": "optional-report-id"
}
```

关键约束：

- 坐标顺序是 `[经度, 纬度]`。
- 顶点数至少 3 个。
- 前端应尽量校验自相交多边形。
- 圈内没有有效点位时不是错误，后端返回简化 Markdown，`point_count` 为 `0`。

## 4. SSE 事件约定

`POST /api/v1/risk-report/stream` 返回 `text/event-stream`。由于它是 `POST + body`，前端必须手动解析流。

| event | data | 前端处理 |
|---|---|---|
| `stage` | `{stage, msg}` | 阶段进度、日志 |
| `reasoning` | `{text}` | AI 思考流，可折叠展示 |
| `content` | `{text}` | AI JSON 输出流，可选展示 |
| `validation` | `{ok, errors}` | 研判校验结果 |
| `a2ui` | `A2uiSurface` | 渲染结构化 UI |
| `warn` | `{msg}` | 非致命降级提示 |
| `done` | `RiskReportResponse` | 最终报告、图件、附件 |
| `error` | `{msg}` | 失败终止 |

事件流以 `done` 或 `error` 结束。

## 5. A2UI 当前组件

后端在 `a2ui` 事件中输出一份扁平组件列表：

```json
{
  "surfaceId": "risk-rpt001",
  "dataModel": {
    "report_id": "rpt001",
    "point_count": 42
  },
  "components": [
    {"id": "risk_card", "type": "card", "props": {}},
    {"id": "chart_type_distribution", "type": "chart", "props": {}}
  ]
}
```

当前可能出现的组件：

| id | type | chartType | 说明 |
|---|---|---|---|
| `risk_card` | `card` | - | 整体风险等级 |
| `hazard_card` | `card` | - | 主要灾害类型 |
| `chart_type_distribution` | `chart` | `pie` | 灾害类型分布 |
| `chart_scale_distribution` | `chart` | `bar` | 规模分布 |
| `chart_warning_distribution` | `chart` | `bar` | 预警等级分布 |
| `chart_hidden_danger_ratio` | `chart` | `pie` | 隐患识别占比 |
| `chart_threat_summary` | `chart` | `bar` | 威胁人数、户数、财产汇总 |
| `chart_trend_distribution` | `chart` | `bar` | 发展趋势状态分布 |
| `chart_trend_forecast_risk` | `chart` | `bar` | 短期预测风险分布 |
| `chart_rain_intensity_distribution` | `chart` | `pie` | 雨情强度分布 |
| `chart_key_point_trend_lines` | `chart` | `line` | 重点点位变形趋势或趋势关注指数折线 |
| `chart_rain_deformation_timeseries` | `chart` | `barLine` | 雨量柱状 + 变形折线双轴图 |
| `key_points` | `list` | - | 重点关注点位表 |
| `recommendations` | `list` | - | 紧急、近期、常规建议 |
| `chart_map` | `map` | - | 地图中心、灾圈边界、聚类凸包、热点格网 |
| `text_limitations` | `text` | - | 数据局限说明 |

前端渲染建议：

- `card`：标题 + 字段键值对。
- `chart`：用 ECharts，根据 `props.chartType` 分发到 `pie`、`bar`、`line`、`barLine`。
- `list`：有 `columns + rows` 时渲染表格；有 `groups` 时渲染分组列表。
- `map`：用 MapLibre 渲染 `center`、`zoneBoundary`、`clusterHulls`、`hotspotGrid`。
- `text`：`format === "markdown"` 时用 Markdown 渲染。

## 6. 最终报告与资源

`done` 事件和同步接口返回：

```json
{
  "report_id": "rpt001",
  "point_count": 42,
  "report_markdown": "...",
  "judgement": {},
  "figures": {
    "map_figure": "/static/figures/rpt001/map_figure.png"
  },
  "attachments": {
    "圈内监测点.csv": "/static/figures/rpt001/圈内监测点.csv"
  }
}
```

前端注意：

- `figures` 和 `attachments` 都是相对 URL，统一 `new URL(rel, API_BASE_URL).href`。
- `report_markdown` 里的图片也是 `/static/...`，渲染前要重写为绝对 URL。
- 在线预览优先用 A2UI 图表；没有 A2UI 或图表不支持时显示 PNG。
- 离线包由后端打 zip，前端只负责下载和保存。

## 7. 版本与批注修订

初次报告自动是 `v1`。前端刷新或恢复会话时调用：

```text
GET /api/v1/risk-report/{report_id}
```

批注修订请求：

```json
{
  "base_version_id": "rpt001_v1",
  "annotations": [
    {
      "block_id": "block-001",
      "block_text": "原段落内容",
      "comment": "请补充诱发因素依据"
    }
  ],
  "mode": "full_markdown"
}
```

修订成功后后端返回完整新版 Markdown，前端直接追加版本并切换预览。失败时不要丢弃本地 open 批注。

## 8. 前端验收清单

对接完成后至少验证这些场景：

- 能提交一个合法多边形并看到 `stage` 进度连续变化。
- 能收到 `a2ui` 事件，并渲染卡片、统计图、趋势图、地图、列表。
- 能收到 `done` 事件，并渲染最终 Markdown。
- Markdown 中的 `/static/...` 图片能正常显示。
- `figures` 图件 gallery 或降级图片能正常打开。
- `point_count == 0` 时展示简化报告，不当作异常。
- 点击下载离线包能得到 zip。
- 能通过 `report_id` 查询版本列表。
- 批注修订成功后能切换到新版本。
- 网络、LLM 或图件降级时能展示 `warn` 或错误提示。

## 9. 当前建议分工

后端侧已可支持前端进入联调。前端建议先完成：

- `riskReportClient.ts`：封装 base URL、SSE 解析、资源 URL 重写。
- `RiskReportPanel`：灾圈输入、生成按钮、进度、错误态。
- `A2uiRenderer`：按 `type` 和 `chartType` 分发组件。
- `MarkdownReportViewer`：Markdown + 图片地址重写 + 大报告渲染保护。
- `ReportVersionPanel`：版本列表、批注状态、修订提交。
- `BundleDownloadButton`：当前版本 zip 下载。

后端后续可继续增强数据质量审计、传感器关联置信度、专项灾种模型，但这些不阻塞当前前端对接。
